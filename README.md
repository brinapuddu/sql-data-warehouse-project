## Welcome to the Data Warehouse and Analytics Project repository! 🚀

This project showcases an end-to-end data warehousing and analytics solution, covering everything from warehouse construction to generating meaningful business insights. Built as a portfolio project, it reflects industry best practices across data engineering and analytics.



## Project Requirements

### Building the Data Warehouse (Data Engineering)

#### Objective
Design and implement a modern data warehouse using SQL Server to centralize sales data, supporting analytical reporting and data-driven decision-making.

#### Specifications

- **Data Sources**: Ingest data from two external systems (ERP and CRM), supplied as CSV files.
- **Data Quality**: Identify and resolve data quality issues before analysis.
- **Integration**: Merge both sources into a unified, analytics-optimized data model.
- **Scope**: Focus exclusively on the most recent dataset; historical tracking is out of scope.
- **Documentation**: Deliver thorough documentation of the data model to serve both business and technical audiences.


### BI: Analytics & Reporting (Data Analytics)

#### Objective
Build SQL-driven analytics to surface actionable insights across:

- Customer Behavior
- Product Performance
- Sales Trends
